# 3D 点云图谱需求 v2.2 稳定基线

## 当前定位

3D 点云未来定位为全系统图谱与运行态的可交互投影入口。

它不是事实源，不直接改写人物、关系、事件、任务、知识、物体、设备、自我状态、目标、证据、行动或反馈。它负责把复杂系统转成可观察、可下钻、可追溯的空间界面。

当前可行路线允许先独立构建完整三维粒子总系统，作为 `Visual World Operating Layer`。该层先通过 mock / projection fixture 验证全局视觉结构和操作意图，待稳定后再把现有人际关系辅助系统作为第一个真实子系统接入。

## 保留内容

- 现有人际关系图谱保持不变。
- 当前事件图谱作为未来全域事件图谱的起点。
- 当前 GUI 悬浮窗、3D 点云、手动旋转、缩放、平移、运行态动画继续保留。
- 当前已构建能力都作为后续总图谱工程的一部分。

## 固定总链路

外部世界 -> 感知层 -> 事件抽取层 -> 全域事件图谱 -> 世界模型层 -> 学习引擎 -> 决策层 -> 行动层 -> 末端安全模块 -> 反馈层。

并行新增横向链路：

外部软件/网络能力 -> 外部能力观察 -> 能力原子 -> 代码/软件能力切片 -> 目标缺口建模 -> 能力拼接计划 -> 实现路径生成 -> 沙盒验证 -> 意识模块评估 -> 需求候选图谱 / 实现候选 -> 可能性预测图谱 -> 决策层 -> 末端安全模块。

各层含义：

- 外部世界：语音、图像、屏幕、位置、文档、网络、设备。
- 感知层：ASR、OCR、视觉识别、场景识别、实体识别。
- 事件抽取层：谁、何时、何地、做了什么、影响了谁、证据是什么。
- 全域事件图谱：社会事件、物理事件、学习事件、实验事件、决策事件。
- 世界模型层：人际图谱、任务图谱、知识图谱、物体图谱、自我状态图谱。
- 学习引擎：知识吸收、类比迁移、因果建模、虚拟训练、物理对齐、内化规则。
- 决策层：目标树、自我意志模型、策略分配、风险审查、资源评估。
- 行动层：沟通、提醒、项目执行、实验设计、设备控制、文档生成。
- 末端安全模块：随意识模块治理的安全范围配置变化而变化；当前沙盒阶段预设为 `sandbox_self_containment.v1`，重点评估自身影响和受控环境。
- 反馈层：结果记录、偏差分析、策略修正、知识更新、意志权重迭代。

## 三层分离

### 事实源

真实人物、关系、事件、任务、知识、物体、设备、自我状态、目标、证据、行动、反馈。

### 投影层

未来 `graph_projection.v1`，只读输出：

- 节点
- 边
- 层级
- `source_refs`
- 摘要
- 证据
- 风险
- 运行态叠加

3D UI 只读投影，不直接读写事实源。

后续独立 3D 总系统可先定义 `graph_projection_vnext`，用来表达多图谱域、能力候选、沙盒验证、可能性预测和安全范围节点。

### 操作意图层

未来 `visual_operation_intent.v1`，只把三维粒子中的点击、选择、下钻、聚焦、候选创建、沙盒验证请求等转换为可审查意图。

UI 不直接写事实、不直接执行外部动作；具体执行由下游模块或适配器决定。

### 展示层

3D 点云、云团、粒子、流光、颜色、下钻、详情面板、相机控制。

展示层可以增强理解，但不能制造不存在的人物、事件、关系或因果。

## 固定图谱域

- 人际图谱。
- 全域事件图谱。
- 任务图谱。
- 知识图谱。
- 物体图谱。
- 自我状态图谱。
- 学习过程图谱。
- 决策过程图谱。
- 行动过程图谱。
- 外部能力转需求图谱。
- 能力拼接与沙盒验证图谱。
- 可能性预测图谱。
- 事实变量融合图谱。
- 意识模块安全范围治理图谱。
- 末端安全审查图谱。
- 反馈过程图谱。

人际图谱保持独立主轴；全域事件图谱不吞并人际图谱，两者通过人物参与事件、事件影响关系、反馈修正关系等边连接。

## 3D 点云交互

默认全局视图，逐层下钻：

1. 全局。
2. 图谱域。
3. 云团或类型。
4. 子实体簇。
5. 单实体。
6. 属性、证据、目标、历史、风险、动作。

用户旋转、缩放、平移、下钻、返回时，运行态叠加不能消失。

## 人际图谱路径

全局 -> 人际图谱 -> 关系类型或关系簇 -> 人物群组 -> 单人物 -> 关系、偏好、目标、关键事件、证据、沟通建议。

## 全域事件图谱路径

全局 -> 全域事件图谱 -> 社会、物理、学习、实验、决策事件 -> 事件链或事件簇 -> 单事件 -> 谁、何时、何地、做了什么、影响谁、证据、后续动作。

## source_refs 约束

投影必须保留来源引用：

- people
- relationships
- events
- tasks
- knowledge
- objects
- devices
- self_states
- goals
- evidence
- raw_inputs
- decisions
- actions
- feedback
- external_capabilities
- requirement_candidates
- forecasts
- variables
- safety_reviews
- capability_slices
- composition_plans
- sandbox_verifications
- safety_scope_profiles

没有 `source_refs` 或投影规则的节点、边、状态高亮，不能显示成已确认事实，只能显示为“待确认推断”或纯 UI 状态。

## 运行态与风险

运行态是叠加层，不写入事实图谱。

风险、权限、dry-run、人工确认、审计引用必须可见。行动层涉及真实外部影响时，不能绕过原有门禁。

风险层级暂定：

- info：普通状态或低风险事实。
- needs_review：需要人工复核的推断、身份、关系或事件。
- blocked：权限不足、证据不足、触发门禁、可能产生真实外部影响。
- danger：涉及真实发送、设备控制、高风险人际关系、隐私或不可逆动作。

## UI 已知项

- 红黄蓝控制线圈应移除，它是 3D 控制器 gizmo，不属于业务图谱语义。
- 放大后必须保持细粒子表达，不能回到粗颗粒球。
- 摄像头手势识别列为未来能力，不作为当前优先实现。

## 开放确认项

同一人物、事件、任务或证据跨多个云团时，当前暂定：

- 底层实体唯一。
- UI 可以多处投影。
- 点击后归并到同一个真实详情。

## 未来拆单

1. UI 小修：移除红黄蓝控制线圈。
2. 投影契约设计：定义 `graph_projection.v1`。
3. 人际图谱接入：把现有人际关系图谱映射为层级点云。
4. 全域事件图谱设计：扩展社会、物理、学习、实验、决策事件。
5. 世界模型接入：人际、任务、知识、物体、自我状态图谱进入全局点云。
6. 过程链路接入：学习、决策、行动、反馈作为过程图谱叠加。
7. 外部能力转需求：网络软件和功能按自身目标上下文重解释为能力候选。
8. 能力拼接与沙盒验证：在无完整匹配软件时，把代码和软件能力切片拼接成实现候选。
9. 可能性预测：基于事实、变量和影响边预测未来分支。
10. 人际辅助系统融合：把当前人际关系辅助系统作为社会认知与人际互动子系统并入大系统。
11. 纯视觉三维粒子总系统：先独立构建 `Visual World Operating Layer`，完成后再接入人际关系辅助系统。
12. 3D 下钻体验：点击、返回、面包屑、详情面板、运行态持续叠加。
13. 性能优化：全局聚合、渐进加载、相关度和风险优先。
