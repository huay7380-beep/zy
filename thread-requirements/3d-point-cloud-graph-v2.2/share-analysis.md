## 当前项目目标边界

当前项目仍然是目标导向型人类社交辅助系统。第一阶段聚焦 B2B 商务沟通和客户跟进，核心目标不是生成一句回复，而是把以下对象连成可验证闭环：

- 用户目标。
- 人物关系。
- 事件线索。
- 决策证据。
- 触发计划。
- 状态上报。
- 反馈回写。
- 审计证据。

因此，任何“全域图谱”“3D 点云”“关系策略分类”的扩展，都不能破坏当前第一阶段闭环，也不能绕过 dry-run、人工确认、权限和审计边界。

## 与当前 v2.2 稳定基线的关系

v2.2 稳定基线已经确认：

- 3D 点云是全系统图谱与运行态的可交互投影入口。
- 人际关系图谱保持不变。
- 事件图谱未来扩展为全域事件图谱。
- 事实源、`graph_projection.v1`、展示层三层分离。
- 每个节点、边、状态高亮必须有 `source_refs`、投影规则或“待确认推断”标记。
- 第一低风险 UI 小修项是移除红黄蓝控制线圈。

本次 share 主题“人际关系策略分类”应被纳入 v2.2 的“人际关系图谱域”和“决策/策略过程图谱”，而不是另起一套独立图谱。

## 对“人际关系策略分类”的保守解构

由于 share 正文不可访问，以下只基于当前项目已有目标、根目录既有人际关系图谱材料、事件图谱材料和用户当前描述进行保守分析。

### 1. 人际关系图谱仍是事实域

人际关系图谱应继续承担：

- 人物实体。
- 关系类型。
- 关系强度。
- 关系阶段。
- 关系健康度。
- 近期互动。
- 关系可见性。
- 关系与事件的接口。

它不应直接承担全部策略决策。策略应在决策层或策略投影层生成，并保留证据引用。

### 2. 策略分类是投影/决策域，不是事实本体

“人际关系策略分类”更适合拆为两类：

- 关系事实分类：例如客户、同事、合作伙伴、朋友、恋人、家人、陌生人、竞争者等。
- 策略建议分类：例如保持低打扰、推进合作、修复关系、降温观察、风险阻断、请求人工确认等。

两者必须分离：

- 事实分类进入人物关系图谱。
- 策略建议进入决策层、行动层或 `graph_projection.v1` 的投影结果。

这样可以避免把“当前应该怎么做”误写成“这个人真实是什么关系”。

### 3. 策略分类必须受目标约束

同一个人物关系，在不同目标下会有不同策略。例如：

- B2B 客户跟进目标：优先看预算、决策链、材料交付、下一步会议、合规边界。
- 关系修复目标：优先看冲突事件、情绪强度、责任归因、沟通窗口。
- 学习/协作目标：优先看知识互补、任务依赖、反馈质量。

因此，3D 点云中“关系云团”不能只按静态关系类型展示，也要支持按当前目标切换策略投影。

### 4. 策略分类必须引用事件证据

策略分类不能只基于人物标签。它必须能回溯到：

- 近期事件。
- 互动频率。
- 事件情绪倾向。
- 帮助、冲突、协作、拒绝、承诺、违约等事件类型。
- 关系权重变化。
- 人工确认记录。

这与当前事件图谱中的 `PARTICIPATED_IN`、`AFFECTS`、`recent_interactions`、`health_score` 思路一致。

### 5. 策略分类在 3D 点云中的显示方式

策略分类不应作为独立大图谱吞并人际图谱，而应作为以下叠加：

- 关系云团的颜色或状态环。
- 人物子云的策略标签。
- 活跃边的推荐方向。
- 详情面板中的可解释策略。
- 风险门禁中的阻断原因。

其中颜色、粒子密度、视觉优先级只是 UI hint，不是事实源。

## 完整目标解构

### 总目标

将当前 3D 点云从“运行状态摘要展示”升级为“全域图谱、关系策略、事件证据、世界模型、学习决策行动反馈闭环”的只读空间投影入口。

该入口既要保留当前项目第一阶段 B2B 社交辅助闭环，又要为未来全域事件图谱和世界模型扩展留出稳定边界。

### 一级目标 A：保持当前项目主线

必须继续保持：

- 第一阶段优先服务 B2B 商务沟通和客户跟进。
- 不自动发送消息。
- 不自动确认高风险事件。
- 外部平台交互默认 dry-run。
- 真实行动需要用户确认、权限检查和审计记录。
- 人际关系图谱、事件记录、决策集群、触发引擎、反馈回写共同形成闭环。

### 一级目标 B：保持人际关系图谱主轴

人际关系图谱继续作为独立事实域，包含：

- 人物。
- 关系。
- 关系类型。
- 关系维度。
- 权重。
- 阶段。
- 健康度。
- 可见性。
- 近期事件引用。
- 与事件图谱的接口。

策略分类只消费这些事实，不覆盖事实。

### 一级目标 C：扩展事件图谱为全域事件图谱

现有事件图谱继续作为起点，未来扩展为：

- 社会事件。
- 物理事件。
- 学习事件。
- 实验事件。
- 决策事件。

每个事件至少保留：

- 谁。
- 何时。
- 何地。
- 做了什么。
- 影响了谁。
- 证据是什么。
- 来源。
- 可信度。
- 风险。
- 是否需要确认。

### 一级目标 D：建立世界模型层

世界模型层由多个图谱并列组成：

- 人际图谱。
- 任务图谱。
- 知识图谱。
- 物体图谱。
- 自我状态图谱。

3D 点云默认展示全局世界模型总览，用户可以进入某一图谱域下钻。

### 一级目标 E：加入学习引擎过程图谱

学习引擎不是事实源，而是“事件如何变成能力和规则”的过程图谱。它应展示：

- 知识吸收。
- 类比迁移。
- 因果建模。
- 虚拟训练。
- 物理对齐。
- 内化规则。

每个学习节点都要引用上游事件、知识或反馈证据。

### 一级目标 F：加入决策过程图谱

决策层应展示：

- 目标树。
- 自我意志模型。
- 策略分配。
- 风险审查。
- 资源评估。
- 关系策略分类。

其中“关系策略分类”应放在决策过程图谱和人际图谱投影之间，既读取关系事实，也读取当前目标和事件证据。

### 一级目标 G：加入行动过程图谱

行动层包括：

- 沟通。
- 提醒。
- 项目执行。
- 实验设计。
- 设备控制。
- 文档生成。

行动节点必须显示：

- 是否 dry-run。
- 是否需要人工确认。
- 权限状态。
- 审计记录引用。
- 阻断原因或通过证据。

### 一级目标 H：加入反馈过程图谱

反馈层包括：

- 结果记录。
- 偏差分析。
- 策略修正。
- 知识更新。
- 意志权重迭代。

反馈应回写到相应图谱或模型层，但 3D UI 只展示回写状态和证据引用，不直接写事实源。

## 3D 点云下钻结构

默认路径：

1. 全局世界模型总览。
2. 图谱域。
3. 云团或类型。
4. 子实体簇。
5. 单实体。
6. 属性、证据、目标、历史、风险、动作。

人际图谱路径：

1. 全局。
2. 人际图谱。
3. 关系类型或策略分类云团。
4. 人物群组。
5. 单人物。
6. 关系、偏好、目标、关键事件、证据、沟通建议。

全域事件图谱路径：

1. 全局。
2. 全域事件图谱。
3. 事件类型。
4. 事件链或事件簇。
5. 单事件。
6. 参与者、时间、地点、行为、影响对象、证据、后续动作。

策略分类路径：

1. 全局。
2. 人际图谱或决策过程图谱。
3. 关系事实云团。
4. 当前目标下的策略分类。
5. 具体人物或关系边。
6. 可解释策略、证据、风险、确认门禁、下一步动作。
