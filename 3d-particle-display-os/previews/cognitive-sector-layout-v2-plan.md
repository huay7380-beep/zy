# 立体认知扇区预览 v2 方案

本预览页用于验证“世界核心为中心”的三维认知排布，不修改原系统代码，只把原有 `zhineng-graph` 的模块语义投射到独立预览中。

## 空间规则

- 中心：世界核心，代表当前需求 OS 的整体认知焦点。
- 角度：模块类型所在扇区。8 个扇区围绕核心形成立体舱室，而不是平面切片。
- 距离：与世界核心/当前目标的相关性。越相关越靠近核心。
- 高度：认知阶段与时间倾向。输入、历史和反馈偏低；状态与行动居中；预测、决策、自我操作偏高。
- 大小和光强：模块重要性。重要性越高，星云核心越大、越亮。
- 流光：动态过程。静态模块不移动，思维流、决策流、反馈流以动态粒子沿关系路径运动。

## 扇区邻接理由

1. 输入感知扇区：外部世界先进入感知与事件抽取，所以邻接证据事件与世界状态。
2. 证据事件扇区：事件承接输入，并沉淀为状态变化，所以位于输入与状态之间。
3. 世界状态扇区：状态和世界模型是共享事实底座，靠近核心。
4. 推理预测扇区：从状态上升到预测、模拟和能力组合，邻接决策治理。
5. 决策治理扇区：汇合推理、策略、安全边界，输出给行动。
6. 行动项目扇区：承接决策，包含行动层和实体工作节点，邻接反馈。
7. 反馈学习扇区：行动结果回流反馈记忆与学习更新，再校准状态。
8. 自我操作扇区：状态对话、视觉 OS、投射契约是观察与接入控制面，靠近反馈和输入，负责把世界状态解释给用户。

## 交互规则

- 拖拽旋转、滚轮缩放，检查真正的三维间距。
- 点击任意星云，在原扇区内展开该模块的子星云。
- 顶部按钮切换总览、思维流、决策流、反馈流和全链路。
- 左侧开关控制扇区舱室、静态模块、标签和三类动态流。

## 影响边界

当前版本只是预览显示逻辑。它不会改变原系统输入、输出、模块 ID、运行态、状态文件或业务流程；未来接入时应保留原 `graph_projection_vnext` / `visual_operation_intent` 契约，仅替换展示层的空间布局算法。
