# 世界核心八扇区认知布局预览方案

这个预览页用于确认理想布局，不连接原系统、不修改原系统状态。

## 是否会影响原系统

如果只在 `3d-particle-display-os/previews` 中生成预览页，不会影响原系统。

如果后续把该布局应用到原系统，影响范围取决于改动层级：

- 只改显示投影坐标：影响较小，原数据、IPC、状态对话、runtime 不变。
- 改 `buildGraphPoints()`：会改变 3D 视觉布局，但不应改变业务状态。
- 改 `WORLD_SYSTEM_NEBULAE` 数据：风险更高，因为会影响星云目录、检查器、状态对话上下文。
- 改 IPC、状态快照、真实图谱读写：风险最高，不属于本布局阶段。

建议先确认预览页，再做隔离迁移，最后才考虑原页面替换。

## 布局规则

以世界核心为中心，八个扇区按人类认知决策闭环排列：

```text
输入 / 感知
→ 证据 / 记忆
→ 世界状态 / 模型
→ 推理 / 预测
→ 决策 / 治理
→ 自我 / 边界
→ 行动 / 工具
→ 反馈 / 学习
→ 输入 / 感知
```

## 三维含义

- 扇区：模块类型和认知职能。
- 半径：与当前目标的相关性。
- 高度：认知状态。
- 粒子大小：重要性。
- 粒子亮度：当前激活度。
- 粒子透明度：置信度。

## 高度规则

- `Y +1.2`：治理、预测、候选、约束。
- `Y 0`：当前目标、当前状态。
- `Y -1.4`：输入、证据、历史。

## 相邻规则

每个扇区的相邻关系都有含义：

- 输入紧邻证据：外部观测必须先成为可追溯证据。
- 证据紧邻状态：证据用于更新当前世界模型。
- 状态紧邻推理：推理必须基于当前状态。
- 推理紧邻决策：预测结果进入候选选择。
- 决策紧邻自我：行动前必须经过边界审查。
- 自我紧邻行动：权限决定能否进入执行。
- 行动紧邻反馈：每个执行必须产生结果。
- 反馈回到输入：结果重新进入观察和记忆。
