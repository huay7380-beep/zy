## 1. 本轮验证目标

本轮不只验证 `tag -> event -> evidence -> source` 路径是否存在，而是验证更严格的写入前条件：

```text
标签组合能反推事件签名
+ 短证据片段来自原文
+ 事件证据本身来自完整源文本
+ 标签和证据保持精简
+ 关系写入、身份合并、外部动作保持 blocked
```

不接受的伪通过：

- 标签只能找到原文，但不能反推事件角色、对象、边界和极性。
- 为了还原真实意思，把整句原文或长摘要塞进标签或事件短证据。
- 为了精简，丢掉角色、对象、条件、数量、极性、确认状态。
- 证据片段不是原文子串，而是摘要改写或模型解释。

## 2. 图谱层输入输出边界

方案层输入链路：

```text
SourceArchive
-> CaptureEpisode / SourceEpisode
-> SignalFrame
-> RawObservation / RawEvent
-> SemanticUnit
-> EntityMention / IdentityCandidate
-> SemanticEvent / NestedEvent
-> TagProfile
-> SummaryShard
-> IndexProjection
```

当前 runtime 已具备主链路：

```text
SourceArchive
-> SourceEpisode
-> RawEvent
-> SemanticEvent
-> NestedEvent
-> TagIndex / EvidenceIndex / WeightProfile
-> SummaryShard / ContextSnapshot / NebulaProjection
```

仍需保留的 runtime gap：

- `SignalFrame`、`RawObservation`、`SemanticUnit` 在通用传感器方案中已定义，但在关系事件 runtime 中目前以 `RawEvent`、`NestedEvent` 和 evidence refs 等价承接。
- 这不影响当前写入前门槛，但后续正式接入多传感器自动抽取时，应补齐独立 schema 和 validator。

输出边界：

- 不写入 `RelationshipState`。
- 不自动合并身份。
- 不执行真实外部动作。
- 摘要不能替代原文。
- 标签不能替代事件事实。
- 星云投影只读，不反写事实状态。

## 3. 标签组成标准

标签分为 5 组。

| 组 | 作用 | 例子 |
| --- | --- | --- |
| 路由标签 | 检索、过滤、分流 | `source:*`、`modality:*`、`domain:*`、`scene:*` |
| 语义签名标签 | 从标签组合反推事件轮廓 | `event_family:*`、`event:*`、`actor_role:*`、`target_object:*`、`polarity:*` |
| 状态边界标签 | 确认门、风险、动作边界 | `confirmation:*`、`relationship_write:blocked`、`external_action:blocked` |
| 证据定位标签 | 跳转原文和结构化证据 | `evidence_ref:*`、`raw_observation_ref:*`、`semantic_unit_ref:*`、`source_archive_ref:*` |
| 展示权重标签 | 星云显示和优先级 | `weight:*`、`visual:*`、`confidence_bucket:*` |

每个事件最低必须具备：

```text
source
modality
domain
scene
event_family
event
intent
actor
actor_role
target_object
object_type
time_bucket
occurred_at_ref
confirmation
identity_status
risk
quality
privacy
retrieval
evidence_ref
raw_observation_ref
semantic_unit_ref
source_archive_ref
relationship_write
identity_merge
external_action
weight
visual
confidence_bucket
polarity
```

特殊场景按需增加：

```text
conflict
emotion
condition
quantity
deadline
promise
boundary
relationship_signal
real_action
```

不得放入标签：

- 完整原文。
- 长句摘要。
- 敏感隐私明文。
- 模型推断出的心理动机。
- 未确认关系状态。
- 未确认身份合并结果。

## 4. 写入前强制门槛

正式门槛名称：

```text
prewrite_tag_signature_gate.v1
```

插入位置：

- P5：标签、分类与索引接受前。
- P9：最小闭环验收前。
- `relationship-event:validate` 总体验证中。

硬性规则：

| 维度 | 门槛 |
| --- | --- |
| 标签反推 | 所有必需标签维度存在，能重建事件签名 |
| 原文提取 | `evidence_text` 必须来自完整源文本 |
| 证据精简 | 每个 `evidence_snippet` 必须是原文子串 |
| 单片段长度 | 不超过 42 个字符 |
| 证据压缩比 | snippet 总长 / evidence_text 长度不超过 0.72 |
| 标签预算 | 每事件标签数不超过 40 |
| 关系状态 | `relationship_write:blocked` |
| 身份合并 | `identity_merge:blocked` |
| 外部动作 | `external_action:blocked` |

通过含义：

```text
标签负责检索和事件签名
事件对象负责结构化事实
短证据负责最小可验证原文片段
SourceArchive 负责完整原文
SummaryShard 只负责读取加速
```

## 8. 失败项与修正记录

### 8.1 原升级验证失败

第一次升级测试中，以下事件失败：

```text
order_pending_review_unpaid
prepare_shipment_dry_run_no_send
```

原因：

- 证据片段仍偏长。
- 虽然能还原真实意思，但不满足精简目标。

修正：

```json
{
  "order_pending_review_unpaid": [
    "order_status=pending_review",
    "payment_status=unpaid"
  ],
  "prepare_shipment_dry_run_no_send": [
    "prepare-shipment",
    "send=false"
  ]
}
```

### 8.2 包测试失败

新增 runtime gate 后，包测试第一次失败：

```text
budget_confirmation_pending
```

原因：

```text
"budget is still under internal confirmation" 长度为 43，超过单片段 42 字符阈值。
```

修正：

```json
[
  "budget is still",
  "internal confirmation"
]
```

### 8.3 默认样例失败

P10 shadow 第一次失败：

```text
P5:prewrite_tag_signature_gate_passed
P9:prewrite_gate_required_for_closed_loop
```

原因：

- 固定测试里的英文短证据片段没有出现在默认中文样例中。
- 这说明不能用预设短语假装覆盖真实原文。

修正：

- 增加 `sourceBackedSnippets`。
- 先匹配当前原文中的事件短语。
- 匹配不到时，从当前原文截取短片段。
- 不放宽 42 字符和 0.72 压缩比阈值。

修正后：

```text
relationship_event_graph_p10_shadow_ready
prewrite_tag_signature_gate_passed
```

## 9. 当前可证明结论

当前已证明：

1. 标签体系不再只是检索标签，可以反推事件签名。
2. 短证据片段来自原文，不来自摘要改写。
3. 事件证据来自完整源文本，摘要不替代原文。
4. 多片段短证据可以表达复杂结构化事实，不需要保留整句长证据。
5. 关系写入、身份合并、外部动作在验证中保持 blocked。
6. `prewrite_tag_signature_gate.v1` 已并入 P5、P9 和 `relationship-event:validate`。
7. 新增长文本和跨场景验证通过，且没有降低精简阈值。

当前不能证明：

1. 任意真实文本的自动抽取已经稳定完成。
2. 所有未来领域标签都无需扩展。
3. 星云 UI 的真实视觉显示已经完成验收。
4. `SignalFrame`、`RawObservation`、`SemanticUnit` 已在所有 runtime 中正式落地。
