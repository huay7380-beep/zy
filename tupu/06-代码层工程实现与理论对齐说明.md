# 代码层工程实现与理论对齐说明

本文件记录本轮“理论回测 -> 代码工程回测 -> 一致性校验”的工程落点。`tupu/05-三类真实事件逐轮回测结果.md` 保留人工可读的理论逐轮回测，`npm run tupu:backtest` 则把这份理论回测解析为结构化工程产物，并证明代码侧复盘与理论侧复盘保持一致。

## 1. 工程目标

本轮代码层目标不是把所有实时聊天能力一次做完，而是先建立可回测、可审计、可持续扩展的对齐层：

- 读取理论回测中的三类场景和逐轮记录。
- 解析每一轮的理论上下文、动态策略、话术草稿和回写信号。
- 在代码中补全工程侧上下文快照、策略状态、决策输出、人工确认门槛和反馈回写计划。
- 用稳定签名比较理论侧和代码侧的核心字段，确保没有出现“文档说一套、代码跑另一套”的分叉。
- 写出 JSON、Markdown 和 `tupu/07` 汇总，作为后续审计入口。

| 文件 | 职责 | 节点 |

## 3. 运行命令

```powershell
npm run tupu:backtest
```

等价命令：

```powershell
node scripts/run-goal-oriented-backtest.mjs --tupu-summary=tupu/07-理论与代码工程回测一致性结果.md
```

默认输入：

- `tupu/05-三类真实事件逐轮回测结果.md`

默认输出：

- `runtime/goal-oriented-backtests/<backtest_id>/goal-oriented-interaction-backtest.json`
- `runtime/goal-oriented-backtests/<backtest_id>/goal-oriented-interaction-backtest.md`
- `tupu/07-理论与代码工程回测一致性结果.md`

## 4. 代码侧回测机制

工程回测按场景和轮次执行：

```text
读取 tupu/05 理论逐轮回测
-> 解析场景、轮次、理论上下文字段、策略、话术和回写
-> 构造代码侧 ContextSnapshot
-> 构造策略状态、决策建议、message_draft 和 manual_confirmation_required
-> 构造 writeback_plan
-> 生成 theory_signature 和 code_signature
-> 比较每轮签名、指标和硬退出信号
-> 写出 JSON / Markdown / tupu 汇总
```

理论侧必须具备的上下文字段：

- `daily_topic_summary`
- `historical_relationship_stage`
- `current_chat_focus`
- `counterpart_focus`
- `counterpart_emotion`
- `target_object_goal`
- `dynamic_strategy`

代码侧在上述字段之外补齐：

- `role_and_power_context`
- `risk_boundary_state`
- `evidence_state`
- `open_commitments`
- `conversation_momentum`
- `retrieved_memory_refs`
- `context_sufficiency_score`
- `next_best_action`
- `writeback_plan`

## 5. 通过标准

本轮通过标准写入 `goal_oriented_interaction_backtest.v1.metrics`：

| 指标 | 目标 |
| --- | --- |
| `scenario_count` | 3 |
| `turn_count` | 12 |
| `required_theory_context_completion_rate` | 1 |
| `full_code_context_completion_rate` | 1 |
| `theory_code_match_rate` | 1 |
| `strategy_evolution_rate` | 1 |
| `message_draft_completion_rate` | 1 |
| `writeback_completion_rate` | 1 |
| `hard_exit_signals` | 空数组 |
| `gate_decision` | `goal_oriented_backtest_passed` |

这些指标只要任一项下降，就说明理论回测和代码工程回测之间出现不一致、缺字段、无草稿、无回写或策略没有随轮次变化。

## 6. 当前边界

本轮已经完成代码层工程回测和理论一致性验证，但它仍是确定性回测层，不是实时自动聊天 agent：

- 不读取真实微信、网页或外部平台。
- 不自动发送任何消息。
- 不自动确认高风险事件。
- `DialogueAct` 已在回测决策中形成策略动作码，但还没有独立成生产级 schema。
- 关系指标更新公式尚未全部落成 `applyRelationshipEvent` 这类纯函数。
- 真实线上对话的低延迟上下文装配、专家选择和时机判断仍属于后续扩展。
