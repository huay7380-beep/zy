## 5. 多学科复审

### 5.1 人类学视角

关注点：

```text
同一行为在不同文化、关系和互动场景中意义不同。
需要区分当事人自己的分类和系统分析分类。
```

新增能力：

- `ContextFrame` 保存互动场景、关系语境、本土标签。
- `emic_label` 与 `etic_label` 分离。
- `interpretation_scope` 约束解释范围。

结论：

```text
anthropology_review = pass_with_review
```

仍需后续注意：真实运行时需要更完整的文化、地域、行业和关系场景标签表。

### 5.2 社会学视角

关注点：

```text
角色、组织位置、权力差异、制度流程和网络结构会影响事件意义。
```

新增能力：

- `TrajectoryRecord` 和 `PhaseSegment` 支持长期社会过程。
- `ContextFrame` 可表达组织/业务/关系场景。
- `PatternClaim` 支持反复行为和长期结构性卡点。

结论：

```text
sociology_review = pass_with_minor_gap
```

保留缺口：正式的 `SocialPositionBinding`、`InstitutionalContext`、`PowerRelationMarker` 仍可作为后续扩展，不阻塞当前叙事层方案。

### 5.3 历史学视角

关注点：

```text
分期、转折、来源版本、资料空白，不能把时间顺序误判为因果。
```

新增能力：

- `PhaseSegment` 解决分期。
- `TurningPoint` 解决转折。
- `SourcePerspective` 解决来源立场。
- `CausalHypothesis` 防止时间顺序误写为因果。

结论：

```text
history_review = pass_with_review
```

保留缺口：后续可单独增加 `SourceGap.v1` 与 `NarrativeVersion.v1`，用于资料空白和叙事版本管理。

### 5.4 叙事学 / 话语分析视角

关注点：

```text
谁在讲述？
以什么框架讲述？
什么被省略？
事件顺序如何被重排？
```

新增能力：

- `SourcePerspective` 记录话语类型和来源位置。
- `NarrativeIndex` 支持长期叙事检索。
- `ContextFrame` 记录解释场景。

结论：

```text
narrative_discourse_review = pass_with_review
```

保留缺口：后续可扩展 `StatementChain.v1`，用于“谁转述了谁”的链路级建模。

### 5.5 法证 / 证据审查视角

关注点：

```text
直接证据、转述、推断、解释必须严格区分。
高风险结论必须回读原文。
```

新增能力：

- 继续使用 `EvidenceAnchor` 作为原文反读基础。
- `SourcePerspective` 区分直接、转述、引用、解释。
- `CausalHypothesis` 保留支持证据和反证。

结论：

```text
evidence_review = pass
```

保留缺口：后续进入真实数据采集时，需要 `EvidenceDerivationRecord`、`ExtractionMethodProfile`、`EvidenceCustodyLog`。

### 5.6 信息科学 / 检索视角

关注点：

```text
长期叙事问题不能只靠人物、标签、时间线粗召回。
需要专门叙事索引和查询意图路由。
```

新增能力：

- `NarrativeIndex` 建立长期轨迹索引。
- `query_intent_type` 区分事实、事件、叙事、趋势、冲突、因果、证据查询。
- `NarrativeContextSnapshot` 明确后续上下文组装格式。

结论：

```text
information_retrieval_review = pass_with_review
```

保留缺口：进入 runtime 后需要验证索引规模、增量更新、失效重建和延迟。

### 5.7 认知科学 / 事件分割视角

关注点：

```text
人类理解事件依赖目标变化、边界、预期违背和状态转移。
```

新增能力：

- `PhaseSegment` 记录阶段入口和退出条件。
- `TurningPoint` 记录 before_state / after_state。
- `PatternClaim` 防止长期模式被随意总结。

结论：

```text
cognitive_event_segmentation_review = pass_with_review
```

保留缺口：后续可把 `goal_state_change`、`expectation_violation`、`event_boundary_confidence` 下沉到 schema 字段。
