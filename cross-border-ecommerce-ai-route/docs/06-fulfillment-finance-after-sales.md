# 订单履约、财务、物流、报关、售后与复购

成交后才是真正的跨境链路。系统目标是让每个订单的证据、责任和下一步清晰可追踪。

## 订单状态机

```text
quote_accepted
-> pi_drafted
-> pi_confirmed
-> payment_pending
-> payment_received
-> production_or_stock_reserved
-> qc_pending
-> qc_passed
-> packing_ready
-> logistics_booked
-> customs_ready
-> exported
-> in_transit
-> delivered
-> after_sales_window
-> repeat_or_close
```

## 生产和备货

| 节点 | 检查 |
| --- | --- |
| 订单确认 | SKU、数量、颜色、包装、定制、交期 |
| 采购/生产 | 物料、排产、产能、关键工序 |
| QC | AQL、功能测试、外观、数量、包装、标签 |
| 客户确认 | 定制样、产前样、包装样 |
| 出货前 | 照片、视频、箱规、装箱单、发票、唛头 |

## 物流方案

| 方式 | 适用 |
| --- | --- |
| 快递 | 样品、小货、高时效 |
| 空运 | 小批量、时效敏感 |
| 海运 LCL/FCL | 大货、成本优先 |
| 铁路/卡航 | 欧亚特定线路 |
| 海外仓 | 复购 SKU、当地交付 |

每个物流方案必须记录：

- 起运地、目的港/门点。
- Incoterms。
- 预计时效。
- 费用构成。
- 抛重/计费重。
- 是否含清关、税费、派送。
- 保险。
- 有效期。

## 报关与单证

常见单证：

- 商业发票。
- 装箱单。
- 合同或 PI。
- 报关单。
- 物流运单/提单。
- 原产地证。
- 检测/认证/许可证。
- 退税联和收汇资料。

AI 可做：

- 单证字段一致性检查。
- HS code 候选提示。
- 唛头/箱规一致性检查。
- 报关资料包清单。

AI 不可做：

- 自动提交报关。
- 自动确认归类和税则。
- 自动承诺目的国清关税费。

## 财务和收汇

| 节点 | 输出 |
| --- | --- |
| PI 确认 | 应收金额、币种、付款路径 |
| 到账 | 银行水单、手续费、汇率、到账差额 |
| 外汇申报 | 贸易背景、订单、报关、合同关联 |
| 发票/退税 | 进项、销项、报关单、收汇、退免税申报 |
| 对账 | 客户维度毛利、渠道维度 ROI |

## 售后

售后不是成本中心，而是复购和产品改进输入。

| 场景 | 处理 |
| --- | --- |
| 到货确认 | 自动询问到货状态和照片 |
| 破损 | 收集照片、箱号、批次、物流信息 |
| 质量问题 | 区分使用问题、运输问题、生产问题 |
| 缺件 | 核对装箱单和 QC 照片 |
| 退换/补发 | 根据责任和合同条款生成方案草案 |
| 好评/案例 | 请求授权使用评价或案例 |

## 复购触发

| 信号 | 触发 |
| --- | --- |
| 预计库存消耗 | 提前 30/15/7 天提醒 |
| 新品匹配 | 推荐同类或升级款 |
| 季节/节日 | 促销或备货提醒 |
| 汇率/原料波动 | 价格有效期提醒 |
| 客户沉默 | 低频维护邮件 |
| 客诉关闭 | 7 天后满意度回访 |

## 审计证据

- `customs_tax_fx_evidence.v1`
