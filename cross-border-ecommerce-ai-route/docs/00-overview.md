# 跨境电商 AI 自动化通路总览

## 一句话目标

把已有产品源头变成一条可被主系统调度的跨境贸易链路：从中国大陆主体合规、产品资料、独立站、获客、询盘、报价、成交、收款、履约、报关、外汇、税务、售后到客户复购，每一步都能被 AI 读取、生成草案、等待人工门禁、执行并回写证据。

## 经营模式定位

建议先采用 `B2B-first + DTC/B2C optional` 的组合。

| 模式 | 适用场景 | 首期策略 |
| --- | --- | --- |
| B2B 独立站询盘 | 自有源头、可定制、客单价较高、买方需要样品/报价/资质 | 主线 |
| B2B 平台引流 | Alibaba、Made-in-China、Global Sources 等站内需求验证 | 辅助获客 |
| B2C/DTC 独立站 | 标品、轻小件、有清晰消费人群 | 待产品适配后开启 |
| 海外仓 | 稳定 SKU、美国/欧盟等区域复购、交付时效敏感 | 第二阶段 |
| 传统外贸 0110 | 大订单、线下合同、一般贸易报关退税 | 与 B2B 并行 |
| 跨境电商 9710/9810 | 线上 B2B 交易、直接出口或海外仓出口 | 按订单和仓配模式选择 |

首期不追求所有渠道同时铺开，先把可验证闭环跑通：`产品资料 -> 独立站 RFQ -> 广告/SEO/社媒获客 -> AI 接待 -> 报价 -> 样品/订单 -> 履约 -> 复盘`。

## 单纯逻辑链

```text
产品源头确认
-> 主体与出口资质准备
-> 产品合规和目标市场准入判断
-> SKU/PIM/成本/库存/交期数据建档
-> 产品图文视频和认证资料制作
-> 独立站和询盘入口上线
-> SEO/Google/LinkedIn/Meta/展会/邮件/平台获客
-> 线索进入 CRM
-> AI 询盘接待和客户画像
-> 需求澄清、样品策略、报价测算
-> 报价单/PI/合同草案
-> 人工确认后对外发送
-> 客户跟进、异议处理、谈判
-> 收款与风控
-> 生产/备货/QC/包装
-> 物流方案和 Incoterms 执行
-> 报关、单证、外汇、税务归档
-> 到货、售后、评价、复购
-> 数据回写、广告优化、客户分层、下一轮触发
```

## 主系统可控结构

本方案把业务拆成 16 个可控节点。每个节点都要满足三件事：

1. 可读：主系统能读取该节点的输入、输出、状态、证据文件和阻塞项。
2. 可控：主系统能触发草案生成、检查、人工确认包、状态推进和审计。
3. 可回写：节点结果能写回 `RawEvent`、`SemanticEvent`、客户/订单/产品图谱和触发计划。

节点列表以 `nodes/process-manifest.json` 和 `nodes/node-catalog.json` 为准。

## AI 自动化边界

| 环节 | AI 可自动化 | 必须人工确认 |
| --- | --- | --- |
| 证照准备 | 生成清单、资料包、状态表 | 实际提交政府/银行/海关/税务/外汇系统 |
| 产品合规 | 生成目标国证书需求矩阵 | 最终 HS code、检测报告、认证、法律判断 |
| 独立站 | 页面草案、产品结构、SEO、表单、FAQ | 上线付款、隐私政策、条款、价格承诺 |
| 获客 | 关键词、广告素材、受众、邮件草案 | 广告预算、真实投放、冷邮件/私信发送 |
| 询盘 | 自动识别需求、补问问题、生成回复 | 首次真实发送和重要客户承诺 |
| 报价 | 成本测算、报价单草案、PI 草案 | 报价有效期、付款条款、Incoterms、发票金额 |
| 履约 | 生产/物流/单证检查清单 | 报关、收款、发货、退款、赔付 |
| 客户维护 | 跟进节奏、复购建议、节日问候 | 对外发送和商业让利 |

## 首期关键指标

| 阶段 | 指标 |
| --- | --- |
| 建站 | 20-50 个核心 SKU 完成英文产品页、RFQ 表单、公司信任资料 |
| 获客 | 每周形成渠道报告：曝光、点击、询盘、CPL、有效询盘率 |
| 询盘 | 首响草案 5 分钟内生成，人工确认后 24 小时内回复 |
| 报价 | 标准品 30 分钟内生成报价草案，定制品 24 小时内生成资料补问和测算草案 |
| 成交 | 每个有效询盘都有 `lead_status`、`next_action`、`next_touch_at` |
| 复盘 | 每周更新 SKU、市场、渠道、话术和报价命中率 |
