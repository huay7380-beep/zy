# 用户获取、获客与推广链路

获客系统的目标不是“发很多内容”，而是稳定获得可报价、可跟进、可成交的有效询盘。

## 渠道组合

| 渠道 | 适合阶段 | 作用 | AI 自动化 |
| --- | --- | --- | --- |
| SEO | 长期 | 产品词、问题词、行业词自然流量 | 关键词、页面结构、内容草案、排名监控 |
| Google Search Ads | 首期验证 | 捕获明确采购需求 | 关键词分组、广告文案、落地页建议、否词 |
| Google Shopping/Merchant | 标品 | 商品流量和再营销 | Feed 生成、问题诊断、图片检查 |
| LinkedIn | B2B | 找采购、分销、品牌、工程/项目客户 | ICP 人群、Lead Gen Form、私信草案 |
| Meta/Instagram | 视觉产品/DTC | 场景种草、再营销 | 素材脚本、受众测试、评论线索识别 |
| TikTok/YouTube Shorts | 演示型产品 | 视频种草和信任 | 视频脚本、标题、标签、剪辑清单 |
| 邮件开发 | B2B | 定向开发采购商/经销商 | 客户清单 enrichment、邮件草案、跟进节奏 |
| WhatsApp | 高意向客户 | 快速确认规格和报价 | 回复草案、资料发送清单、节奏提醒 |
| 展会/线下名片 | 行业成交 | 高质量客户和代理商 | 名片识别、会后邮件、报价提醒 |
| B2B 平台 | 需求验证 | 平台内询盘和背书 | 产品发布、回复模板、线索同步 |

## ICP 客户画像

每个目标市场先定义 ICP，不要直接全世界投放。

| 维度 | 示例 |
| --- | --- |
| 国家/地区 | 美国、德国、阿联酋、澳大利亚、东南亚 |
| 客户类型 | 分销商、品牌商、工程商、进口商、零售商、亚马逊卖家 |
| 决策人 | Owner、Purchasing Manager、Sourcing Manager、Category Manager |
| 采购触发 | 新品扩充、供应商替换、价格优化、定制、交期问题 |
| 核心顾虑 | 质量、认证、MOQ、交期、付款安全、售后、进口税 |
| 首次转化物 | 目录、报价、样品、规格书、认证包、案例 |

## 获客漏斗

```text
曝光
-> 点击/访问
-> 产品页浏览
-> 资料下载/聊天/RFQ
-> 有效询盘
-> 需求澄清
-> 报价
-> 样品/小单
-> 大货/复购
```

## SEO 内容集群

| 页面类型 | 内容 |
| --- | --- |
| Product page | SKU、规格、MOQ、交期、应用 |
| Category page | 类目对比、选型指南 |
| Application page | 行业应用场景 |
| Comparison page | 材质/型号/工艺/成本比较 |
| FAQ page | 付款、物流、认证、定制、样品 |
| Compliance page | 目标市场证书和标签说明 |
| Case page | 客户案例、出货案例、项目案例 |

## 广告验证节奏

首期按 4 周循环：

1. 周 1：目标市场和关键词小预算测试。
2. 周 2：根据搜索词、询盘质量和跳出率调整页面。
3. 周 3：扩展有效词组，添加再营销和 Lead Form。
4. 周 4：复盘 CPA、有效询盘率、报价率、样品率。

## 冷启动开发信

开发信不应群发同一模板。AI 先做客户研究，再生成 3 类草案：

- `problem_based`：针对客户当前产品线或采购痛点。
- `product_fit`：强调我们的产品与其渠道匹配。
- `sourcing_alternative`：强调质量、交期、成本或定制替代。

人工确认前不得真实发送。

## 渠道事件回写

每个渠道都必须输出：

- `campaign_id`
- `source`
- `audience`
- `offer`
- `landing_page`
- `spend`
- `impressions`
- `clicks`
- `leads`
- `qualified_leads`
- `quotes`
- `orders`
- `notes`

这样主系统才能判断继续投放、暂停、改页面、改报价或改产品。

## 首期推广最小组合

如果资源有限，建议：

1. 10-20 个核心产品页 + 类目页。
2. Google Search 小预算验证高意图关键词。
3. LinkedIn 搜索目标客户，手工确认后定向触达。
4. 1 个行业下载资料或选型表作为 Lead Magnet。
5. 展会/名片/老客户资料统一导入 CRM。
6. 每周输出渠道复盘报告。
