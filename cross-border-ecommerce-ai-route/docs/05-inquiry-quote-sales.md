# 询盘接待、报价与销售链路

询盘处理的核心是：快速判断真假和价值，补齐报价缺口，给出可信报价，并持续推进下一步。

## 询盘分层

| 等级 | 特征 | 响应 |
| --- | --- | --- |
| A | 公司/邮箱可信、产品明确、数量/目的地/交期清晰、采购周期近 | 立即澄清缺口并生成报价草案 |
| B | 需求明确但信息不全 | 先补问 3-5 个关键问题 |
| C | 泛泛询价、无公司信息、数量很小 | 提供目录/起订量/样品政策，低成本跟进 |
| D | 垃圾、钓鱼、明显不匹配 | 标记并停止自动触达 |

## AI 接待流程

```text
接收询盘
-> 识别语言和客户身份
-> 抽取 SKU、数量、目的国、交期、定制、目标价
-> 匹配 ProductMaster 和 PriceBook
-> 检查合规、库存、MOQ、交期
-> 判断缺口问题
-> 生成首响草案
-> 人工确认
-> 发送
-> 记录客户反馈
-> 生成报价草案或下一轮问题
```

## 首响原则

首响必须做到：

- 感谢并确认收到。
- 复述客户需求，证明我们读懂了。
- 若可报价，说明会提供报价和预计时间。
- 若信息不全，只问最关键问题，不一次问太多。
- 保持专业、简短、可执行。

## 报价输入

| 输入 | 来源 |
| --- | --- |
| SKU/型号 | 客户询盘、产品匹配 |
| 数量 | 客户提供或 AI 补问 |
| 目的国/港口 | 客户提供或 AI 补问 |
| 包装 | 产品默认或客户定制 |
| 交期 | 生产/库存数据 |
| 成本 | 产品主数据和采购/生产成本 |
| 运费 | 物流报价或历史模板 |
| 付款条款 | 客户等级和风控策略 |
| Incoterms | EXW、FOB、CIF、DAP/DDP 等 |
| 有效期 | 通常 7-15 天，视原料和汇率 |

## 报价输出

标准报价草案应包含：

- 客户公司和联系人。
- 产品型号、图片、规格。
- 数量阶梯价。
- 币种。
- Incoterms 和交付地点。
- 包装方式、箱规、毛净重。
- 交期。
- 样品费用和样品交期。
- 付款条款。
- 报价有效期。
- 备注：认证、定制、运费假设、税费不含/含。

## 报价门禁

AI 生成报价后必须检查：

- SKU 是否真实存在。
- 成本是否有来源。
- 毛利是否低于红线。
- 交期是否有生产/库存依据。
- 运费是否带时效和有效期。
- Incoterms 是否与报价金额一致。
- 认证/合规承诺是否真实。
- 付款条款是否符合客户风险等级。
- 汇率和币种是否明确。

任何一项不通过，输出 `quote_blocked`，不允许真实发送。

## 跟进节奏

| 时点 | 动作 |
| --- | --- |
| 报价后 24 小时 | 确认是否收到，问是否需要调整数量/运输方式 |
| 报价后 3 天 | 提供样品/案例/证书，推动下一步 |
| 报价后 7 天 | 询问项目状态，提供有效期提醒 |
| 报价后 14 天 | 标记冷却，进入月度维护 |
| 客户打开邮件/下载资料 | 触发高意向提醒 |

## 异议处理

| 异议 | AI 响应方向 |
| --- | --- |
| 价格高 | 比较材质、质检、包装、交期；提供数量阶梯和替代方案 |
| MOQ 高 | 提供样品单、小批量试单、混款方案 |
| 交期长 | 分批发货、现货款替代、空运样品 |
| 不信任 | 提供工厂视频、认证、第三方验货、贸易保障 |
| 运费高 | EXW/FOB/CIF/DAP 多方案对比 |
| 需要独家代理 | 先以试销目标和区域销售额门槛谈判 |

## 成交输出

当客户接受报价，进入：

- `proforma_invoice_draft.v1`
- `sales_contract_draft.v1`
- `payment_instruction_draft.v1`
- `production_order_request.v1`
- `fulfillment_plan.v1`

这些文件必须人工确认后才能对外发送或执行。
