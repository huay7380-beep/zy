# 运行状态笔记

## 当前角色

本文件定义运行状态接口，只负责记录运行状态、节点、次数、错误和摘要。人物、关系、事件、反馈、索引和审计的业务存储不写入 `runtime/state/`，应按 `docs/12-人物与事件存储方案.md` 落到 `data/` 和审计日志。

## 目标

运行状态笔记用于把智能体的当前状态暴露给外部系统和人工检查者。它不是普通日志，而是状态管理接口。

## 文件位置

```text
runtime/state/current-status.json   机器可读的当前状态快照
runtime/state/operator-note.md      人类可读的运行笔记
runtime/state/run-events.jsonl      追加式事件流
```

## current-status.json 字段

| 字段 | 含义 |
| --- | --- |
| `schema_version` | 状态结构版本 |
| `project` | 项目名 |
| `status` | `idle`、`running`、`completed`、`failed` |
| `updated_at` | 最近更新时间 |
| `run_count` | 累计运行次数 |
| `current_run_id` | 当前运行 ID |
| `current_node` | 当前节点 |
| `node_counts` | 每个节点累计运行次数 |
| `active_runs` | 尚未结束的运行 |
| `last_run` | 最近一次完成或失败的运行摘要 |
| `recent_errors` | 最近错误 |
| `metrics` | 成功、失败、节点事件等统计 |

## 事件类型

```text
run_started
node_started
node_completed
run_completed
run_failed
```

## 外部读取方式

外部系统可以直接读取：

```powershell
Get-Content runtime/state/current-status.json
Get-Content runtime/state/operator-note.md
Get-Content runtime/state/run-events.jsonl -Tail 20
```

如果并发压测、进程中断或外部写入导致快照和事件流不一致，可以使用事件流重建快照：

```powershell
node scripts/rebuild-state.mjs
```
