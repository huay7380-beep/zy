# 人类社交辅助系统集成方案

## 定位

系统从“沟通策略智能体”升级为“人类社交辅助系统”时，上层目标不再只是生成一句话，而是：

```text
基于用户目标、人物关系、历史事件、日常行为准则和当前约束，帮助用户规划后续社交进程。
```

这里的“社交”同时覆盖：

- 商业接洽：客户、合作伙伴、上级、下属、项目协作者、常驻商家等。
- 社会生活：朋友、邻居、家长圈、亲密关系、家庭关系、社区服务等。

## 两份图谱文件的职责

### 人际关系图谱

`人际关系图谱完整版.md` 是人物与关系的基础层，提供：

- 关系类型：77 类关系，覆盖自我、亲缘、亲密、地缘、社会契约、负向博弈。
- 关系策略：沟通风格、敏感话题、维护频率、信任度、披露深度、行为准则。
- 事件接口：每类关系常见事件、高影响正负事件、关系转换触发条件。
- 动态规则：复合关系、场景切换、季节性事件、关系转换、三角关系检测。

### 事件图谱

`事件图谱-基于人际关系图谱.md` 是事件与线索的记录层，提供：

- 事件节点：记录时间、参与人、类型、重要性、情感、证据、可见性。
- 事件边：人物参与边 `PARTICIPATED_IN`，事件影响关系边 `AFFECTS`。
- EPP 管道：入口过滤、场景切分、候选事件提取、事件分级归并、确认入库。
- 事件检索：通过人物、关系、时间、类型、线索、影响边回放历史。

## 新增系统层级

```text
用户初始目标
  -> 目标解释层
  -> 人物关系上下文层
  -> 事件线索检索层
  -> 日常行为准则层
  -> 进程安排层
  -> 沟通策略与表达层
  -> 执行复盘与事件入库层
```

## 关键原则

1. 数据与策略分离：图谱记录事实、关系、事件；策略层只读取摘要和约束。
2. 事件不是聊天记录：聊天、语音、日历只是信号载体，事件必须有参与者、影响、时间锚点和因果指向。
3. 用户确认优先：P1 主事件、敏感关系变化、高风险规划不自动入库或自动执行。
4. 不做隐蔽操控：系统可以帮助用户更清晰、更合规、更体面地安排互动，但不能利用对方脆弱性、隐私或误导对方。
5. 商业和生活共用底座，但行为准则不同：商业场景优先契约、证据、边界；生活场景优先关系温度、长期信任、适度维护。

## 进程安排模型

进程安排不是单条建议，而是一个可跟踪计划：

```json
{
  "plan_id": "plan_001",
  "initial_goal": "推动客户进入技术评审",
  "scene": "business",
  "primary_person_id": "person_client_a",
  "relationship_context": {},
  "event_context": {},
  "constraints": [],
  "steps": [
    {
      "step_id": "step_1",
      "objective": "先降低采购压力",
      "recommended_action": "发送低承诺评审邀请",
      "channel": "wechat",
      "timing": "24小时内",
      "required_evidence": ["最近一次预算异议", "技术负责人未参与"],
      "risk_notes": ["不直接催签", "不制造虚假紧迫感"],
      "success_signal": "对方确认评审或说明真实卡点",
      "fallback": "若未回复，48小时后发送价值提醒"
    }
  ],
  "review_questions": []
}
```

## 商业接洽规划逻辑

商业场景优先考虑：

- 关系类型：客户、合作伙伴、项目协作者、上级、下属、供应商等。
- 契约边界：合同、费用、交付、责任、保密、验收。
- 事件线索：会议、报价、异议、投诉、签约、违约、交付。
- 行为准则：书面确认、证据留存、不过度承诺、不泄露商业机密。

默认输出：

```text
目标拆解 -> 关键人判断 -> 历史事件线索 -> 当前阻力 -> 下一步进程 -> 沟通策略 -> 风险边界 -> 复盘入库
```

## 社会生活规划逻辑

社会生活场景优先考虑：

- 关系类型：朋友、家人、邻居、家长圈、常驻商家、社区工作人员等。
- 关系温度：健康度、互动频率、情感趋势、近期事件。
- 日常行为准则：尊重边界、不过度索取、礼尚往来、保护隐私。
- 事件线索：帮助、冲突、节日、生日、道歉、邀请、拒绝、长期无互动。

默认输出：

```text
关系状态 -> 近期事件 -> 维护必要性 -> 合适动作 -> 语气边界 -> 后续观察 -> 是否记录新事件
```

## 与当前智能体链路的接入点

现有链路：

```text
任务创建 -> 上下文结构化 -> 对象建模 -> 关系判断 -> 目标拆解 -> 阻力诊断 -> 技巧匹配 -> 策略卡 -> 表达生成 -> 安全校准 -> 对话模拟 -> 复盘记忆
```

```text

  -> 目标拆解
```

## 需要补齐的缺口

1. 两份图谱文件缺少可直接执行的完整 JSON schema；当前已有 `schemas/social-graph.schema.json`，但仍缺独立 `RawEvent`、`SemanticEvent`、`StorageAudit`。
2. 人际关系图谱定义了关系类型，但用户实例层 `Person`、`RelationshipEdge` 仍需要按 `docs/12` 落地到本地存储。
3. 事件图谱定义了 EPP，当前已有轻量规则提取，但还缺少场景切分、归并、冷却期、线索追加和存储回写。
4. 事件图谱提到 `策略关系定义.md`，当前工作区没有该文件，应先以关系图谱中的 `default_strategy` 作为策略依据。
5. 隐私规则需要进入运行时，而不能只停留在文档。
6. 社交计划必须有用户确认节点，避免系统自动放大错误关系判断。

## 第一版落地范围

第一版只做：

- 关系类型核心知识库。
- 事件类型核心知识库。
- 用户图谱输入 schema。
- 规则型事件候选提取。
- 关系/事件上下文摘要。
- 基于目标的进程安排。
- 接入现有沟通策略工作流。

暂不做：

- 自动导入聊天、社交媒体、通讯录。
- 自动写入高风险事件。
- 自动发送消息。
- 复杂图数据库和 3D 可视化。
