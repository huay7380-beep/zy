## 当前缺口

- 已有 `data/` 目录和人物/事件存储运行时骨架。
- 已有独立的 `RawEvent`、`SemanticEvent`、`StorageAudit` schema 文件。
- 已有 Obsidian 可视化流程树：`views/obsidian/system-process-tree.canvas`。
- 已有 Obsidian Markdown 显化流程树：`views/obsidian/system-process-tree.md`。
- 已有 MVP 批量试点门禁入口：`node scripts/run-mvp-loop.mjs --all`。
- 已有试点样本导入入口：`node scripts/import-pilot-records.mjs --input=examples/pilot-import-batch.sample.json`。
- 已有导入样本驱动 MVP 闭环入口：`node scripts/run-mvp-loop.mjs --pilot-import=examples/pilot-import-batch.sample.json`。
- 已有 MVP 应用内报告页入口：`node scripts/run-mvp-loop.mjs --pilot-import=examples/pilot-import-batch.sample.json --write-report`。
- 已有用户专项测试反馈入口：`node scripts/run-mvp-loop.mjs --pilot-import=examples/pilot-import-batch.sample.json --user-feedback=examples/mvp-user-feedback.sample.json --write-report`。
- 已有 MVP 自代理预检入口：`node scripts/run-mvp-self-agent-cycle.mjs`，会刷新导入样本闭环、报告页、MVP 完成度审计、流程树同步校验、轻量 MVP 压力测试、外部输入包、安全模板和外部输入就绪报告，输出 `runtime/self-agent-preflights/<preflight_id>/mvp-self-agent-preflight.json`、Markdown 摘要、`runtime/input-kits/<kit_id>/mvp-external-input-kit.json`、`runtime/input-templates/<template_init_id>/mvp-external-input-templates.json` 和 `runtime/input-readiness/<readiness_id>/mvp-external-input-readiness.json`。
- 已有 MVP 外部输入安全模板入口：`node scripts/init-mvp-external-input-templates.mjs` 或 `npm run mvp:inputs:init`，可在不重跑完整自代理预检时读取最新输入包，重生成 `runtime/user-inputs/templates/**` 可编辑模板和 `runtime/input-templates/<template_init_id>/mvp-external-input-templates.json`，但不会写入真实目标文件。
- 已有 MVP 外部输入就绪校验入口：`node scripts/validate-mvp-external-inputs.mjs` 或 `npm run mvp:inputs:check`，默认读取最新输入包并聚合检查 PT-003/PT-004 真实材料状态。
- 已有 PT-003 真实试点样本材料交接入口：`node scripts/prepare-pt003-pilot-materials.mjs` 或 `npm run mvp:pt003:prepare`，读取最新 readiness 与只读桌面 observation，输出 `runtime/pt003-pilot-materials/<material_id>/pt003-pilot-materials.json`、Markdown 报告和 `pilot-import.real.draft.json`，但不写入真实目标文件、不执行真实发送。
- 已有 PT-004 受控平台快照生成入口：`node scripts/prepare-controlled-platform-snapshot.mjs` 或 `npm run mvp:pt004:prepare`，写出 `runtime/user-inputs/platform-snapshot.real.html` 和 `runtime/user-inputs/platform-snapshot-preview.real.json`。
- 已有 MVP 真实输入受控试跑入口：`node scripts/run-mvp-real-input-trial.mjs` 或 `npm run mvp:real-trial`。根目录 `npm run mvp:real-trial` 默认接入 `examples/mvp-user-feedback.sample.json`，用于生成 `user_test_review`、`second_pass_optimization` 和反馈报告契约；真实用户反馈可通过底层脚本 `--user-feedback=<file>` 替换。缺材料时停止在报告层，材料 ready 后运行完整闭环、报告、完成度审计和流程树同步，输出 `runtime/real-input-trials/<trial_id>/mvp-real-input-trial.json` 和 `mvp-real-input-trial-report.html`。
- 已有 MVP 统一状态看板入口：`node scripts/write-mvp-status-dashboard.mjs` 或 `npm run mvp:status`，汇总自代理、目标审计、真实输入试跑、完成度审计、流程树同步、压测和运行状态，输出 `runtime/status-dashboards/<dashboard_id>/mvp-status-dashboard.html`。
- 已有 MVP 总目标逐项审计入口：`node scripts/audit-mvp-objective.mjs` 或 `npm run mvp:objective:audit`，默认读取最新自代理预检和完成度审计，输出 `runtime/objective-audits/<audit_id>/mvp-objective-audit.json`。
- 已有 MVP 完成度审计入口：`node scripts/audit-mvp-completion.mjs`，输出 `runtime/audits/mvp-completion-audit.json` 和 `runtime/audits/mvp-completion-audit.md`。
- 已有 MVP 全链路压力测试入口：`node scripts/stress-mvp-loop.mjs --runs=10`，输出 `runtime/mvp-stress-tests/<stress_id>/mvp-stress-test.json` 和 Markdown 摘要。
- 已有流程树同步校验入口：`node scripts/validate-process-tree.mjs`，输出 `runtime/process-tree-validations/<validation_id>/process-tree-validation.json` 和 Markdown 摘要。
- 已有桌面 ContextSnapshot 决策试跑入口：`node scripts/run-desktop-context-bridge.mjs` 或 `npm run desktop:context`，可把一条或多条桌面 `IntakeObservation` 自动转成生成版 `PilotImportBatch` 和 `ContextSnapshot`，再输出并行专家矩阵、理论预测、独立审查和 `message_draft`。
- 已有平台 dry-run 连接器契约：`schemas/platform-dry-run-connector.schema.json`，可验证测试页或页面快照预览到达、草稿存在和发送阻断。
- 已有平台快照验证入口：`node scripts/validate-platform-snapshot.mjs --snapshot=examples/platform-snapshot.sample.html --preview=examples/platform-snapshot-preview.sample.json`，输出 `runtime/platform-snapshot-validations/<validation_id>/platform-snapshot-validation.json`。
- 已有目标导向图谱工程回测入口：`node scripts/run-goal-oriented-backtest.mjs` 或 `npm run tupu:backtest`，读取 `tupu/05`，输出 `runtime/goal-oriented-backtests/<backtest_id>/goal-oriented-interaction-backtest.json` 和 `tupu/07-理论与代码工程回测一致性结果.md`，用于证明理论逐轮回测与代码工程回测一致。

## 演进路线

### V0 可运行骨架

- 固定 B2B 场景
- 规则型节点
- 状态笔记
- 基础测试
- 压力测试入口

### V1 标准 MVP

- 5 个高频 B2B 子场景
- 30 张技巧卡
- 多版本表达
- 异议处理
- 简单复盘
- 基础记忆确认机制

### V2 个人沟通教练

- 用户风格分析
- 表达问题诊断
- 对话模拟训练
- 能力评分
- 长期改进建议

### V3 团队版

- 团队话术库
- 成功案例库
- 失败案例库
- 新人训练库
- 客户异议看板

### V4 企业级平台

- CRM、企业微信、飞书、邮件接入
- 自动识别沟通机会
- 自动生成会议后跟进
- 组织级沟通资产和效果看板
