# 智能体构建方案

## 当前角色

本文件是早期“沟通策略智能体”的运行工作流说明，负责定义可观察、可测试的节点编排。系统升级为“人类社交辅助系统”后，本文件不再单独决定第一阶段全局范围。

- 上层系统目标以 `docs/05-人类社交辅助系统集成方案.md` 为准。
- 第一阶段执行与验收以 `docs/10-第一阶段执行方案.md` 为准。
- 人物与事件存储以 `docs/12-人物与事件存储方案.md` 为准。
- 文档之间的互证关系以 `docs/13-文档互证审计与修复清单.md` 为准。

## 定位

第一版智能体不是“大而全自动 agent”，而是一个稳定、可观察、可测试的沟通策略工作流。

核心链路：

```text
任务创建 -> 上下文结构化 -> 对象建模 -> 关系判断 -> 目标拆解 -> 阻力诊断 -> 技巧匹配 -> 策略卡 -> 表达生成 -> 安全校准 -> 对话模拟 -> 复盘记忆
```

## 总架构

Workflow Orchestrator

## 子智能体职责

| 节点 | 职责 | 第一版实现 |
| --- | --- | --- |
| 场景识别 | 将模糊输入归入标准沟通场景 | 规则匹配 |
| 上下文结构化 | 把聊天、邮件、纪要转成事实、时间线、异议、未知项 | 规则抽取 + LLM 预留 |
| 对象建模 | 判断对方角色、权力、风险顾虑、证据偏好 | 角色模板 |
| 关系判断 | 判断冷启动、初步兴趣、阻力、停滞、防御 | 关键词和信号 |
| 目标拆解 | 拆最终目标、阶段目标、本轮目标、最小成功 | 目标阶梯模板 |
| 阻力诊断 | 识别价格、价值、信任、时机、权限、风险、内部协同等阻力 | 规则分类 |
| 技巧匹配 | 根据场景、角色、目标、阻力选择技巧组合 | 知识库映射 |
| 策略卡 | 输出局势、目标、策略、表达框架、避免项 | 模板生成 |
| 表达生成 | 生成专业版、强推进版、温和版、短消息版 | 模板生成 |
| 安全校准 | 拦截虚假、冒充、胁迫、隐藏重大事实等风险 | 边界规则 |
| 对话模拟 | 模拟积极、拖延、异议、拒绝等反应 | 规则模拟 |
| 复盘记忆 | 生成可写入的学习项，不直接污染长期记忆 | memory patch |

## 状态笔记要求

状态笔记是运行时一等能力，必须满足：

1. 外部可读：JSON 快照和 Markdown 笔记都放在 `runtime/state/`。
2. 可追踪：每次运行有 `run_id`，每个节点有开始、完成、失败事件。
3. 可统计：记录累计运行次数、节点运行次数、错误次数、最近一次运行耗时。
4. 可调试：记录当前节点、活动运行、最近错误和最近输出摘要。
5. 可扩展：未来可接入 Web 控制台、CLI、监控系统或数据库。

## 安全边界

允许：

- 基于真实事实进行更清晰、更有利、更有节奏的表达。
- 使用低承诺推进、选择题推进、风险前置、证据补强、异议澄清等技巧。
- 提醒真实机会成本和真实业务风险。

禁止：

- 编造客户案例、数据、身份、资质、背书。
- 冒充他人或伪造沟通来源。
- 制造虚假紧迫感。
- 隐瞒重大费用、风险、限制或违约条件。
- 威胁、羞辱、恐吓、情绪勒索。
- 利用未成年人、老人、病人、经济困难者等明显脆弱群体。

## 第一版验收标准

1. 用户粘贴一段 B2B 沟通上下文后，系统能输出结构化局势。
2. 系统能给出本轮目标，而不是直接生成催促话术。
3. 系统能识别主要阻力，并给出至少 3 个技巧组合。
4. 系统能生成策略卡和 3 个表达版本。
5. 系统能模拟至少 3 种对方反应。
6. 系统每次运行都更新状态笔记。
7. 测试能验证工作流、状态笔记和安全校准。
